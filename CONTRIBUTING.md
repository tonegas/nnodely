## Getting Started

1. **Fork** the repository and clone your fork:

   ```bash
   git clone https://github.com/<your-username>/nnodely.git
   cd nnodely
   ```

2. **Install dependencies**:

   ```bash
   pip install -r requirements.txt
   ```

3. Create a new branch for your work:

   ```bash
   git checkout -b my-feature
   ```

> If you are one of the maintainers follow the [branching and commit
> strategy](https://github.com/tonegas/nnodely/BRANCHING_COMMITS.md).

## Reporting Issues

If you encounter a bug, have a question, or want to propose an improvement:

- Check existing issues to avoid duplicates
- Open a new issue using **one of our templates**.
- For bugs, include minimal steps to reproduce if possible

## Feature Requests

If you want to propose a big new feature, please do so by opening a
[discussion](https://github.com/tonegas/nnodely/discussions). For small
features you can open an issue (make sure they are small!).

---

## Pull Requests

We follow a lightweight PR process:

1. Keep PRs **small and focused**.
2. Use one of our
   [templates](https://github.com/tonegas/nnodely/docs/PULL_REQUEST_TEMPLATE)
3. Any PR should address an issue directly, so link to it.
4. Make sure tests pass (and add tests when relevant).

Once submitted, your PR will be reviewed as soon as possible.
